# 多 Agent 儲能競爭市場 — 設計與結果

> B 家族(多 agent + λ 競爭)的地基與核心機制。搭配 `new_src/agents/v1_single.py`(單 agent)、
> `new_src/agents/v2_multi.py`(多 agent 引擎)。價格死(price-taker,歷史價當錨),
> λ 是把 agent 綁在一起的膠水,SoC 是唯一會流動的狀態。**結算窗 = 一週**(SoC 跨天連續)。

---

## 1. 兩個關鍵觀念(建之前先校正)

**觀念 1:「回合」不是時間流逝,是求均衡的迭代。**
真實市場同一小時的拍賣**只出清一次**,所有 agent **同時**投標。用**輪流最佳反應
(iterated best response = best-response dynamics)**解那次拍賣的均衡:大家互相反應到沒人想改
(收斂 = Nash 均衡)。「回合上限」= agent 只往前想幾步(**有限理性**,沒收斂就停)。
正式理論靠山:位勢賽局保證 best-response 收斂(Monderer-Shapley 1996)、
連續策略凹賽局 Nash 存在(Rosen 1965)。詳見 `LITERATURE.md` B 區。

**觀念 2:這一切只因為 λ 才成立。**
agent 2 的決定憑什麼被 agent 1 看到?只有當**動作會改變價格**——也就是 λ。
沒 λ(純 price-taker):互不影響 → 一輪就收斂、退化成各做各的(計分板)。
有 λ:大家一起賣壓價 → 才有真正互動、才需要好幾回合。

## 2. 市場機制

**結算窗 = 一週(168h)**:SoC 在一週內連續累積、週末歸零。比每日歸零真實——
晚上充的電隔天早上放得掉,不再有「跨天漏財」的假象。

**單一 agent(`agents/v1_single.py`)** — 一顆電池(容量 `E`、功率 `P`、往返效率 `η`)當 price-taker:

```
看到的價 = 歷史 DA 價                    # price-taker,不動價
報酬     = Σ 價 ×(放電 − 充電)          # 放電賣、充電買
SoC 走一遍整週:充電存 RT·c、放電抽 d/RT,夾住 [0, E]   (RT = √η,效率只在這裡算一次)
```

參考策略:`perfect`(完美預知,LP 最佳排程 = 天花板)、`naive`(固定 hour-of-day = 地板,
時段由 `naive_hours` 從**訓練期平均價格輪廓**學,不是手挑;k=4 配合電池 4 小時容量)。

**多 agent(`agents/v2_multi.py`)** — 加 λ 價格衝擊 + 輪流最佳反應:

```
看到的價_i = 歷史價 + λ ×(其他 agent 的淨買量)     # λ = 膠水
每回合輪流每個 agent i:排程_i = perfect(看到的價_i)  # 對別人最佳反應 = 重用同一個 LP
重複到收斂或到回合上限
統一出清價 = 歷史價 + λ ×(所有 agent 的淨量)         # uniform price,含全體衝擊
```

`perfect()` 直接重用 → 多 agent 只是單 agent 地基上的一層薄殼。

## 3. 結果

**v1 單 agent 地基(電池 4 MWh / 1 MW / η=0.9,真實 DA 價,測試期後 30%,週窗):**

| 區  | 完美預知(天花板) | naive(地板) | naive 捕獲率 |
| --- | ---------------- | ----------- | ------------ |
| DK1 | €120,698/年      | €50,585/年  | **42%**      |
| DK2 | €127,889/年      | €48,405/年  | **38%**      |

天花板→地板中間那段,就是「會預測/會決策值多少錢」——預測器與多 agent 都在搶這段。
(註:naive 時段改成資料驅動後,捕獲率從舊版手挑的 27% 升到 ~40%;週窗也修好了 DK2 跨天漏財。)

**v2 多 agent 競爭(示範週 2023-01-02,DK1,週內尖峰原始價 €195,10 顆同質電池):**

| λ      | 回合 | 尖峰出清價 | 尖峰壓低 | 每家報酬/週 | 報酬消散 |
| ------ | ---- | ---------- | -------- | ----------- | -------- |
| **0**  | 2    | €195       | €0       | €2,180      | 0%       |
| 1      | 4    | €185       | €10      | €1,722      | 21%      |
| 3      | 4    | €165       | €30      | €1,117      | 49%      |
| 6      | 4    | €142       | €53      | €617        | 72%      |
| **12** | 14   | €125       | €70      | **€41**     | **98%**  |

**故事:**

- **λ=0**:純 price-taker,互不影響,各賺 €2,180/週。
- **λ↑**:10 家搶著尖峰放電 → 自己把尖峰價壓下去(€195→€125)→ 每家越賺越少。
- **λ=12**:競爭過度,租值幾乎消散殆盡(98%),且要 14 回合才勉強收斂(玩家多、耦合強)。

這是 Nash 均衡在動:agent 互相反應、把利潤競爭掉。印證理論也印證文獻(`LITERATURE.md` A4:
**同質**電池頭對頭常不是均衡、把價差壓垮)。

> ⚠️ **這張表是機制示範,不是結果。** 上面的 λ=1~12 是人為灌大的值,真實估計是
> **λ=0.004**(§3.9)。**真正的旋鈕是體量不是 λ**——λ×淨量才是關鍵,`λ=12 × 10MW`
> 等價於 `λ=0.004 × 30GW`。要看真實 λ 下的競爭強度請看 §3.5(結論:不咬)與
> `SIMULATOR_OVERVIEW.md` §3.2 的體量掃描。舊版把 λ 灌到 100× 當作「競爭崩盤」的
> 證據,那其實是在偷偷把體量做大。

   **真正的旋鈕是體量,不是 λ**——λ×淨量才是關鍵,`λ=0.4 × 10MW ≡ λ=0.004 × 1GW`。
   呼應 C1:市場力屬於少數大組合,不屬於幾千台 price-taker 小電池。
5. **月窗 ≈ 週窗**:結算窗從 168h 拉到 720h,每週折算只多 ~1%(SoC 能跨週搬運的價值很小)。
   排名完全不變 → 週窗是夠用的、便宜的選擇。
6. **預測誤差的成本 >> 價格衝擊的成本**:v2.1(100%)→ v2.2(93%)掉 **7pp** = 不知道未來價
   的代價;而 v2.1 距單顆基準 **0pp** = 別人跟你搶的代價。**預測準度是唯一的瓶頸。**
   而且 v2.2 λ=0.004(93%)**等於**單顆 LightGBM(93%):真實 λ 下 10 家一起競爭對每家
   毫無傷害,結果完全退化回「一顆有預測誤差的電池」。要 λ=0.4(≡ 1GW 車隊)兩種成本
   才開始疊加(v2.2 82% < v2.1 89%)。
7. **異質才有互動**:上面全部是同質 agent(同預測、同策略),所以每家報酬必然相同——
   那是對稱性不是發現。改成異質後(3 LightGBM / 3 Ridge / 4 naive)才問得出「誰賺最多」,
   而且發現**預測優勢在競爭下被放大 6 倍**。見 `SIMULATOR_OVERVIEW.md` §3.3。

- **誠實星號**:λ 為觀察性估計(多變數偏導數),非因果識別;嚴謹版該用風當工具變數(未做)。
- **下一步**:體量 × 預測品質的二維掃描;真體量接 Energinet BRP。

### 修過的一個真 bug(天花板虛高)

舊 `perfect()` 的 LP 只管**小時結束時** SoC ≤ E,不管「充完當下」有沒有爆容量,
所以在**負電價**小時它會「同一小時先充爆、再放掉」偷分——但 `settle()` 會把超額充電夾掉,
於是 LP 帳面 €3,484、實際只結算到 €3,130。症狀:**競爭 agent 的報酬 > 天花板(最高 107%)**,
因為它們的排程剛好被夾得比較少。修法:LP 補上 `soc_{t−1} + RT·c_t ≤ E`,
讓 **LP 可行集 == settle 可行集**(夾取永不觸發)。修完沒有任何一列超過 100%,
天花板本身也升到真值(週 €172,029 → €175,359)。`market.demo()` 留了負價 self-check 守住它。

## 3.6 v3 = Cournot(agent 內化自身衝擊)— `nash(...)` / `solve_day(..., br=cournot_br)`

v2 的每家是 price-taker:對「別人推移後的價」最佳反應,但不管**自己**一動也會推移出清價。
v3 讓每家內化自身衝擊 = 最佳反應從 LP 升級成 QP:

> max Σ seen·(放−充) **− λ·w_i·Σ(充−放)²**

多的二次罰項 = 「我賣越多、把自己的價壓越低」的邊際成本。體量 `w_i` 越大罰得越重 → 大玩家自制。

**沒裝 QP solver**:QP 的可行集 == `perfect()` 的 LP polytope,所以用 **Frank–Wolfe** 迭代——
線性化後的 oracle 剛好是 `perfect(有效邊際價 = seen + 2λw_i·(充−放))`,閉式線搜,重用現成 LP。
λ=0 時退回純 LP(sanity)。

**結果(λ=0.004 結構估計,8 週,10 家異質體量,`experiment.py v3`)**:

| 體量                | 策略        | 最大玩家 | 最小玩家 | 大小差  | 車隊/單顆基準 |
| ------------------- | ----------- | -------- | -------- | ------- | ------------- |
| 10 MW(丹麥實況)     | price-taker | 100%     | 100%     | 0pp     | 10.0×         |
|                     | Cournot     | 100%     | 100%     | 0pp     | 10.0×         |
| 1 GW                | price-taker | 90%      | 90%      | 0pp     | 898×          |
|                     | Cournot     | 90%      | 90%      | 0pp     | 899×          |
| **10 GW**(機制可見) | price-taker | **36%**  | 39%      | **3pp** | 3,775×        |
|                     | **Cournot** | **40%**  | 42%      | 1pp     | **4,098×**    |

**三個發現:**

1. **真實尺度下 Cournot 與 price-taker 沒有差別。** 10 MW 和 1 GW 兩組完全重疊——
   `λ·w_i` 太小,自我蠶食還沒開始咬,內化與否無所謂。**這是誠實的負面結果。**
2. **10 GW 才修掉「大玩家反而最慘」的 artifact**:price-taker 的大玩家 36% < 小玩家 39%
   (它不自制、把自己的價壓垮)。Cournot 大玩家升到 40%,和小玩家 42% 幾乎並排(3pp → 1pp)。
3. **車隊總報酬變高**(3,775× → 4,098×,+8.6%):大家都克制 → 少了互相砸價的內耗。
   教科書 Cournot vs 完全競爭的結論。

(舊版此處是「λ=3.7 下 16%/25% → 27%/29%」。方向相同,但那個 λ 是灌大 900 倍的值;
換成結構 λ 後,同樣的機制要 10 GW 車隊才浮現。)

## 3.7 v4 = 風力情境(大家有風機,部分裝電池)— `wind_scenario(...)`

一群同體量風電商,其中一部分額外裝一顆電池。**風力用真實 day-ahead 預測出力**
(Energinet 陸域+離岸,見 `DATA_CATALOG.md` #3),**價格用真實價**(已內含全體風力 →
風的價格衝擊不重複算);唯一的新衝擊 = 電池套利的 λ×淨量。

> 為什麼堅持用真實風力:早期版本拿「價格的反向」當風力 shape 的代理。那會把
> **「風與價負相關」這個待驗證的性質變成建構出來的假設**,外部性的符號就不可信了
> (而外部性的符號正是這節的主要結論)。改用真資料後純風商收益 4,140 €/MW/週,
> 反推容量因數 ~41%,符合丹麥陸域(~30%)+離岸(~45%)的混合。

**掃「多少比例裝電池」(DK1 全 63 週,λ=0.004,`experiment.py v4`):**

| 體量                | 裝電池家數 | 純風商 €/MW風/週 | 電池溢價 €/MW電/週 | 外部性 €/MW風/週 |
| ------------------- | ---------- | ---------------- | ------------------ | ---------------- |
| **10 MW**(丹麥實況) | 0 → 10     | 4,140 → 4,140    | 2,163 → **2,161**  | 0.0 → **0.1**    |
| **1 GW**            | 0 → 10     | 4,140 → 4,153    | 2,100 → **1,877**  | 0.0 → **12.8**   |
| **10 GW**           | 0 → 10     | 4,140 → 4,260    | 1,609 → **678**    | 0.0 → **120.0**  |

**兩個發現(方向與舊版一致,但門檻在體量):**

1. **電池溢價隨採用率消散——但要體量夠大才看得到**:10 MW 時 2,163 → 2,161(消散 0%,
   彼此搶不到對方的價差);1 GW 時 −11%;10 GW 時 **−58%**。
2. **電池對純風商是正外部性**:風與價負相關(風多時便宜),電池充在「便宜=風多」時段抬升
   低價 → 純風商的風賣得更好。同樣要體量:0.1 → 12.8 → 120 €/MW風/週。
   **符號在三個尺度上都是正的**:儲能滲透率↑會緩解風電的自我蠶食。

## 3.8 三把尺 C / N / M + 勾結指標 — `cartel(...)`

> **先正名**:前面的「天花板 / 佔天花板」其實是**單顆基準(per-battery numeraire)**——「1 顆電池、λ=0」
> 的完美預知收益,當**計量單位**用。它**不是車隊的上限**:10 顆各拿 99% → 車隊 27,552 €/週 =
> **9.9× 單顆基準**。作為單位可比,作為上限在多家世界不成立。以下 % 一律是「佔單顆基準」。

同一個世界(10 顆電池、同一條 `act`、λ>0)有三種決策方式,聯合利潤必然 **M ≥ N ≥ C**:

| 尺           | 誰決策              | 內化 λ 衝擊?    | 對應                  |
| ------------ | ------------------- | --------------- | --------------------- |
| **C** 競爭   | 很多小廠 price-take | ~0(廠數→∞)      | `competitive()`       |
| **N** Nash   | 各自 best-respond   | 只內化自己 λw_i | v3 Cournot(10 異質廠) |
| **M** 卡特爾 | 一個規劃者統管全隊  | 內化全體 λ·ΣW   | `cartel()`            |

`cartel()` = 「own weight = 總體量」的 Cournot,一個 QP 解完。`competitive()` = 把總體量拆成
n 個對稱小廠跑 Cournot(廠越多 → 每廠市場力→0 → 越接近 price-taking);用 Gauss-Seidel 逐一更新
自帶阻尼 → 穩定、聯合利潤 ≥0(見下方「乾淨 C」)。

**結果(λ=0.004 結構估計固定,只動體量;4 週;單位=車隊聯合利潤 ÷ 單顆基準,`experiment.py scales`):**

| 體量                | C 競爭   | N Nash   | M 卡特爾 | 勾結空間 (M−N)/N |
| ------------------- | -------- | -------- | -------- | ---------------- |
| **10 MW**(丹麥實況) | 10.0×    | 10.0×    | 10.0×    | **0.0%**         |
| **1 GW**(北歐 2030) | 900.1×   | 900.2×   | 901.6×   | **0.2%**         |
| **10 GW**           | 3,923.7× | 3,969.8× | 4,397.3× | **10.8%**        |

**勾結指標**:Δ = (Π_observed − Π_N) / (Π_M − Π_N)。Δ=0 = 靜態 Nash、Δ=1 = 完全卡特爾;
Δ∈(0,1) 且無通訊 = **tacit collusion 且被量化**(對接 Calvano Q-learning 定價文獻)。M 是缺的**上錨**。

**三個發現:**

1. **真實尺度下沒有戰場**:10 MW 時 C=N=M=10.0×,三把尺**塌成一點**(勾結空間 0.0%)。
   連全知卡特爾都賺不到比競爭多的錢 → **tacit collusion 這題目在丹麥現況下不存在**。
   這不是壞消息,是一個乾淨的結論:**丹麥現在的儲能規模沒有市場力可言。**
2. **勾結空間要體量才張得開**:1 GW 時仍只有 0.2%,要到 **10 GW 才有 10.8%**。
   所以要研究 tacit collusion,實驗必須設在 GW 級並在論文裡講清楚那是**前瞻情境**而非丹麥現況。
3. **Nash 幾乎已是完全競爭**:即使 10 GW,C=3,924 ≈ N=3,970(差 1.2%),而 N→M 有 10.8%。
   單邊克制拿不到什麼,**真正的勾結空間全在「協調」= N→M 那一段**。
   (⚠️ 舊版曾寫「C→N 吃掉大半」,那是壞掉的 naive C 造成的假象;修好後 C≈N。)

**⚠️ 校準問題(寫論文要揭露)+ 已修的一個 benchmark bug:**

- **✅ 已修:乾淨 C(`competitive()`)**。舊 C 用「零內化的天真 price-taker」(優化看 `act`、結算卻付
  `act+λ·全體` 的衝擊)→ 高 λ·ΣW 時過度倒貨、競爭利潤**倒賠**(real DK1 大尺度曾 −5%/−56%,理性
  price-taker 不該虧錢)。試過 Walrasian tâtonnement(bang-bang 震盪)、representative-agent(發散)、
  ½-penalty QP(小 λ 過度自制)全不行;**正解 = 拆成多個對稱小廠跑 Cournot**,Gauss-Seidel 自帶阻尼 →
  穩定 ≥0。修後 real DK1:全隊 10MW→C98%、1GW→45%、10GW→2%,不再倒賠。(順帶 `_cournot_br` 的 FW
  以 duality-gap 收斂、iters 上限 100 → 大 λ·ΣW 也不欠收斂;M 在 ~1–3GW 隊穩,10GW 超尺度才需更多 iter。)
- **λ 可能被低估(雙重計算)**:`act` 已是真實出清價、已被既有套利者(挪威水力/CHP/現有 BESS)壓平。
  λ 在**殘餘**價差上估、卻當「從無到有的衝擊」用 → 偏低。丹麥儲能極小 → `act`≈pre-storage → 偏差二階,
  但方向要講明:100% 主因是「1 MW 真的邊際」,真正旋鈕是**體量**不是 λ。
- **λ 的識別問題(見 §3.9)**:現用的是多變數偏導數(觀察性),控制了鄰國/燃料/邊界,
  但非因果識別。嚴謹版該用**風當工具變數**。舊版的單變數 OLS 高估 8 倍,已修。
- **p₀ = `act` 的循環仍在**:基準價是歷史出清價,裡面已含所有人的策略行為。所以本模型
  可以做**相對比較**(A agent vs B agent,兩者面對同一個 p₀),但**不能宣稱絕對市場力**
  (那需要「全員出邊際成本」的競爭反事實,得從成本結構建 fringe,不在本專案範圍)。

## 3.9 λ 從哪來 — `agents/fringe.py`

**λ = 0.0042**(DK1 中位)= 多變數偏導數 ∂p/∂x:控制德/瑞殘餘、燃料、邊界容量、日曆、氣溫後,
本地殘餘負載對價格的偏效果,在**每小時自己的狀態向量**上求值,取中位數當模擬器的純量。

**為什麼偏導數才是對的。** λ 要回答的是「**我的電池多買 1 MW,價格漲多少**」——那個實驗裡
德國的風、天然氣價、瑞典的水**都不變**,正是 ceteris paribus = 偏導數。而單變數 OLS 斜率
回答的是另一個問題:「當本地殘餘負載剛好高 1 MW 時(**通常是因為德國也剛好缺電**),價格
高多少」,那裡面混著鄰國效應。

| 估計方式                   | λ           | 控制了什麼               |
| -------------------------- | ----------- | ------------------------ |
| 單變數 OLS(**舊版用的**)   | 0.034       | 無                       |
| 單變數,分燃料 regime       | 0.010–0.044 | 只有燃料                 |
| **多變數偏導數(現在用的)** | **0.0042**  | 鄰國+燃料+邊界+日曆+氣溫 |

**高估 8 倍的來源是混淆**:丹麥風大的日子北德通常也風大(同一片天氣系統),所以 DK1 殘餘
負載低時德國殘餘也低、價格也低,但**不是因為丹麥本地的供需**。單變數把德國的效果全記到
本地頭上。這也用合成資料驗證過(`_demo_confounding()`:真值已知的世界裡單變數高估 9 倍,
控制鄰國後收斂回真值)。

**穩定性**:有限差分步長 150–800 MW 之間,中位數只在 0.0037–0.0048 動(±20%),
不動搖「比舊值小一個量級」這個結論。DK2 = 0.0082,約 DK1 兩倍(市場更小,同樣 MW 推動更多)。

**同時發現的兩個坑**(都寫在 `fringe.py` 的 `_DROP` 註解):

1. **共線性**:x = 負載 − 風 − 光,三者的預測值原本都在特徵集裡。「固定它們、只動 x」是
   矛盾的反事實,樹在那個方向沒有資訊 → 偏導數假性趨近 0(第一次跑出 0.0004)。
2. **樹是階梯函數**:有限差分步長 50 MW 時 42% 的小時導數為 0(落在同一葉子),要用 300 MW。

**λ(x) 的形狀(診斷用,未接進模擬器)**:保序迴歸估的 p₀(x) 顯示曲棍球桿——主體區間
λ 在 0.01–0.03,x>2,400 MW 才翹到 0.12。但**燃料維度的變異更大**(gas 低/中/高 = 0.010/0.019/0.044,
跨 4.4 倍)。之所以沒把 λ(x) 接進模擬器:它是**局部**斜率,GW 級車隊會沿曲線跑遠,
`p = p₀ + λ(x)·Q` 的線性化會失準,要改成 `p = p₀(x+Q)`(非線性,會動到 `perfect()` 的 LP)。
而且只 condition 在 x 上會忽略更大的燃料維度 → 看似精確、實則只做了一半。

⚠️ **觀察性估計,非因果識別。** 控制了可觀察的混淆,沒處理不可觀察的。嚴謹版該用**風當
工具變數**(風外生、只透過殘餘負載進價格)——資料齊全,尚未做。
