## 1. `perfect` 是什麼?為什麼叫「天花板」?(LP 說明)

**LP = 線性規劃(Linear Programming)**:在一堆線性限制下,找讓某個線性目標最大的解。標準工具
(scipy 的 HiGHS)幾毫秒解出「全域最佳」,不是逼近、不是啟發式——就是**數學上證明過的最好解**。

我們的電池套利寫成 LP:

```
決策變數:每小時充多少 c_t、放多少 d_t(一週 168 小時 → 336 個數字)
目標(要最大):Σ 價_t ×(d_t − c_t)          ← 貴時放、便宜充,賺價差(線性)
限制:
  0 ≤ c_t ≤ P(功率上限)                    ← 一次充放不超過 1 MW
  0 ≤ d_t ≤ P
  0 ≤ SoC_t ≤ E(電量上限)                   ← 存量不爆容量、不透支
  SoC_t = SoC_{t−1} + √η·c_t − d_t/√η        ← 電量隨充放走,效率在這裡算
  SoC_窗末 = 0                                ← 週末電放光
```

**「perfect」= 完美預知**:它拿到的是**真實的未來價**(事後諸葛),所以它的排程是「這一週最多能賺
多少」的**上限**。任何策略(預測、競爭)都不可能贏過它——這就是為什麼它是所有 % 的分母(天花板)。
(修過一個 bug:LP 以前少了「充電當下不爆容量」的限制,在負電價週會虛高;已修,見 `MULTI_AGENT_MARKET.md`。)

## 2. 為什麼 perfect / 預測模型都只有「一顆電池、一個玩家」?

**對,因為它們是「預測值多少錢」的問題,不是「競爭」的問題。** 這一組全部是:

- **單顆電池**、**price-taker**(自己買賣不動市場價,沒有 λ)。
- 差別只在「它以為未來價長怎樣」:
  - `perfect` 看到真實價(作弊,當上限)
  - `LightGBM/Ridge/Lasso` 看到自己的**預測價**
  - `naive-24h` 用「昨天同一小時的價」當預測
  - `naive 固定時段` 根本不預測,每天固定時段充放(當下限)
- 全部用**同一個 LP** 排程,再用**真實價結算**(預測錯 → 賣錯時段 → 被罰)。

它回答的是:**「我的預測拿去交易,能吃到天花板幾成?」** 沒有第二個玩家,不需要 λ。

**同質 vs 異質差多少?** 幾乎沒差(下表),因為真實 λ 下每家都是 price-taker,體量不改變每單位報酬。
差異只在把 λ 灌大、競爭變兇時才浮現。

**競爭又分 v2.1 / v2.2 兩種資料前提**(排程時「信」什麼價,結算都用真實出清價):

| 版本     | agent 排程依據      | 隔離的成本              |
| -------- | ------------------- | ----------------------- |
| **v2.1** | 真實價(上帝視角)    | 純**價格衝擊**          |
| **v2.2** | **LightGBM 預測價** | 價格衝擊 + **預測誤差** |

v2.1 假設完美預知、只為隔離 λ;v2.2 才寫實(agent 只能靠自己的預測排程)。下面 §3.2 兩者都列。

### 3.1 競爭配置:每單位報酬佔天花板(全 63 週,DK1)

| 配置       | λ=0.037(真實)    | λ=0.37(10×) | λ=3.7(100×)          |
| ---------- | ---------------- | ----------- | -------------------- |
| 同質 10×1  | 99%(range 99~99) | 90%(90~90)  | 39%(**range 38~40**) |
| 異質 10 家 | 99%(range 99~99) | 90%(90~90)  | 39%(**range 36~40**) |

**怎麼讀:** 真實 λ 下 10 家各拿 99%,大小玩家一模一樣(所以給單一數字是誠實的)。只有把 λ 灌到
100× 競爭才咬得動,**這時大玩家每單位反而賺最少**(異質:大玩家 36% < 小玩家 40%)——它一動就
壓垮自己的價(自我蠶食)。這是「agent 不內化自身衝擊」的 artifact,Cournot 策略(QP)會修掉它。

**v2.1 → v2.2 掉多少 = 預測誤差的價:** 真實 λ 下 v2.1 99% → v2.2 92%(−7pp),而 v2.1 距天花板只
1pp。「不知道未來價」害你 7pp、「別人搶」只害你 1pp——**預測準度是主要瓶頸**。且 v2.2(92%)≈
單顆 LightGBM(93%):真實 λ 太小,競爭幾乎退化回一顆有預測誤差的電池。

**兩把尺別混:** rMAE = 預測準度(分母 = naive-24h 的誤差);佔天花板 = 錢(分母 = perfect 的 €)。

## 4. 電池設定:哪些有根據、哪些是簡化?(誠實版)

| 設定            | 值                             | 有根據嗎?                                                        |
| --------------- | ------------------------------ | ---------------------------------------------------------------- |
| 功率 P / 容量 E | 1 MW / 4 MWh(= **4 小時電池**) | ✅ 有根據。「4 小時時長」是併網電池的**業界標準規格**,不是亂猜。 |
| 往返效率 η      | 0.90                           | ✅ 合理。鋰電池併網系統典型 85–92%。                             |
| 結算窗          | 一週(168h)                     | ✅ 已驗證月窗≈週窗,週窗夠用。                                    |

**但有一個大簡化,我之前沒對你講白:「把鯨魚當成電池」本身就不寫實。**

丹麥真正的大玩家**不是電池**,是**風電 + 生質/垃圾 CHP + 燃煤**的組合(§5)。丹麥的併網電池**很少**
(最大的案子才 10–30 MW,整個北歐到 2030 預估才 ~1,800 MW)。所以「10 顆電池競爭」是研究「套利 +
價格衝擊」的**乾淨玩具模型**,不是丹麥發電的字面寫照。

**這代表什麼:** 我們的 λ 是「價格對淨買量的斜率」,它**已經把真實市場**(含發電商、跨國連接線)
壓縮成一個數。所以模型作為「儲能套利者面對一個有價格衝擊的市場」是站得住的;但如果論文要宣稱
「模擬 Ørsted/Vattenfall 的市場力」,就得誠實升級:把鯨魚改成**帶邊際成本報價的發電商**,
或明講「電池 = 抽象的策略性套利者」。**這是要跟指導教授確認的定位問題,不是程式 bug。**

## 5. 丹麥真實數據(有出處,非幻想)

### 5.1 發電結構(2022,佔發電量)

風電 **54%**、生質+垃圾 **23%**、燃煤 **13%**、太陽能 **6.3%**、天然氣+油 **3.8%**。總發電 ~35 TWh。
兩大發電商:**Ørsted、Vattenfall**(維基未給精確 MW/市佔——**精確體量數字仍是 TODO**,
需查 Energinet/DUR 的 BRP 清單)。

### 5.2 併網電池:很少

最大案例約 10 MW(Better Energy Hoby)、30 MW/43 MWh(EWII Bornholm)、200 MWh 光儲(European
Energy Kvosted)。北歐到 2030 預估 ~1,800 MW。→ **電池不是丹麥的價格制定者。**

### 5.3 跨國連接線 = DK 價格的真正主導(你賭對了)

DK1/DK2 尖峰需求才幾 GW,但連接線容量遠大於此——**價格常常是被進出口流、而非本地發電決定的**:

| 從      | 到   | 容量                                      | 備註                |
| ------- | ---- | ----------------------------------------- | ------------------- |
| **DK1** | 德國 | 1,780 MW(南向)/ 1,500(北向)AC             | 最大單一連接        |
| DK1     | 挪威 | 1,700 MW(Skagerrak 1–4)                   | 挪威水力 = 靈活調節 |
| DK1     | 瑞典 | 740 MW(Konti-Skan)                        |                     |
| DK1     | 荷蘭 | 700 MW(COBRAcable, 2019)                  |                     |
| DK1     | 英國 | 1,400 MW(Viking Link, 2023;目前限 800 MW) | 世界最長陸海 HVDC   |
| **DK2** | 瑞典 | 1,700 MW(Øresund)                         |                     |
| DK2     | 德國 | 600 MW(Kontek)                            |                     |

**結論(§5.3):** 光 DK1 對外就有 ~6,000+ MW 連接容量,遠超本地尖峰需求。德國風光、挪威水力、
瑞典核電/水力,透過這些線直接壓進 DK 價格。**所以你的直覺對:德/挪/瑞(+荷/英)強烈影響 DK 電價。**
好消息是我們的 λ(從真實 DK residual–price 斜率估)**已經內含**這些跨國效應——它是 reduced-form,
把「跨國套利會平抑價格」這件事一起吸收了。要更細就得顯式建模連接線(v3+ 再說)。

## 6. 版本規劃與結果

**v1 單電池** ✅ — perfect(天花板)/ naive(地板)/ 預測模型交易,單顆 price-taker。

**v2 多 agent** ✅ — 10 家(同質 + 異質)在 λ 價格衝擊下輪流最佳反應解 Nash。分兩種資料前提:

- **v2.1**(上帝視角,對真實價排程):真實 λ=0.037 只削 1%(99% 天花板)。
- **v2.2**(對 LightGBM 預測價排程、真實價結算):掉到 92% ≈ 單顆 LightGBM 93% → **預測誤差成本
  (7pp)>> 價格衝擊成本(1pp)**。詳見 §3.2 與 `MULTI_AGENT_MARKET.md` §3.5。

**v3 = Cournot** ✅ — agent 內化自身衝擊,最佳反應 LP → **QP**(收益多 `−λ·w_i·(自己淨量)²`)。
沒裝 QP solver:用 Frank–Wolfe 重用現成 `perfect()` LP 當 oracle。**結果(λ=3.7):修掉「大玩家最慘」
artifact(v2 大 16% vs 小 25% → v3 大 27% vs 小 29%),且車隊總報酬升 27%→33%**(自制 → 少內耗)。
細節見 `MULTI_AGENT_MARKET.md` §3.6。

**v4 = 風力情境** ✅ — 一群同體量風商,部分裝電池。風用 day-ahead 預報 shape、價用真實價
(風的衝擊不重複算),電池套利用 λ 競爭。**結果(λ=0.037):電池溢價隨採用率上升而消散(2,682→2,268
€/MW電/週),且電池對純風商是正外部性(+21 €/MW風/週 @ 全採用,因風與價負相關 −0.47)**。
細節見 `MULTI_AGENT_MARKET.md` §3.7。
