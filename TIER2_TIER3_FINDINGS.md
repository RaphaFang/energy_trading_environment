# Tier-2(跨境)+ Tier-3(燃料)資料與發現

> 本檔記錄 DK1/DK2 電價預測專案中,ENTSO-E 跨境特徵 + 燃料價的**資料內容**與**這輪對話的所有發現**。
> 搭配 `DATA_CATALOG.md`(6 源總表)、`MODEL_MATH.md`(模型算式)、`new_src/data/entsoe_features.py`、`new_src/data/fuel_prices.py`、`new_src/data/load_duckdb.py`。

---

## 0. 一句話總結

在本地 5 源(電價/天氣/出力預報/residual/日曆)之上,加了**鄰居殘差 + 跨境容量 + 燃料價**,
把 LightGBM 的 rMAE 從 **0.76 → 0.54**(DK1)、**0.70 → 0.53**(DK2),比 naive 地板少錯 46–47%。
而且驗證過**無 data leak**——模型最倚重的特徵正好是這輪建的德國殘差與燃料價。

---

## 1. 核心觀念:為什麼「只看鄰居風光」不夠

一開始 Tier-2 只抓了鄰居的**風光發電預測**。使用者點出這是**半截訊號**:

> 德國風大,不代表會把便宜電倒進丹麥——要看德國**自己的用電有沒有吃掉**。

正解是**殘差(residual / net position)**:

```
鄰居殘差 = 隔日負載預測 − 隔日風光預測
         = 需要多少「可調度電源(火力/水力/進口)」來補
```

- 殘差**負**(風光 > 用電)→ 電多到滿出來 → 倒向丹麥 → DK 價崩 / 負電價
- 殘差**高**(無風 + 高需求)→ 得動用貴的天然氣 → DK 價高

同樣的風光量,搭配不同的用電,結果完全相反。**殘差才是「淨出口壓力」。**

---

## 2. 火力發電怎麼進模型(merit order)

**發電機分兩類,只有一類有隔日預報:**

| 類型                 | 例子           | 出力由誰決定           | 有隔日預報?         |
| -------------------- | -------------- | ---------------------- | ------------------- |
| 靠天(must-run)       | 風、光         | 老天                   | ✅ 有(TSO 必須預報) |
| 可調度(dispatchable) | 煤、氣、核、水 | **市場**(價夠高才開機) | ❌ 沒有             |

所以「德國明天燒多少煤/氣」**沒有**隔日預報——那是競價的**結果**(內生),不是原因。
去預報它 = 用答案猜答案。火力靠**兩個外生量**間接捕捉:

```
殘差(需要多少火力 = 量)  ×  燃料價 TTF/CO2(火力多貴 = 價)  ≈  高價時段的價格水準
```

- 2022 天價 = 天然氣暴漲(價,€10→€339)撞上無風(量)。
- 德國核能已於 **2023-04** 全部關閉。

---

## 3. 不是每個鄰居都有效(用資料證明)

「鄰居殘差 vs 丹麥電價」的相關強度差很多:

| 鄰居           | 相關係數   | 殘差波動(std) | 為什麼                                     |
| -------------- | ---------- | ------------- | ------------------------------------------ |
| **德國 DE_LU** | **+0.451** | 13,460(巨大)  | 補殘差靠**火力**(貴又會變)→ 殘差一動價就動 |
| 瑞典 SE_4      | +0.232     | 725           | 水力/核 baseload 厚,殘差被壓住             |
| 瑞典 SE_3      | +0.118     | 2,134         | 同上,波動小 = 資訊少                       |

**結論**:德國殘差是主力;瑞典殘差價值低。**瑞典管的是「地板」,該用 Tier-3 水庫水位(water value),不是殘差。**
(概念:挪威/瑞典水力可蓄水 → 用不用現在發電是「機會成本」→ 水庫水位設定北歐電價地板;風光管波動,水庫管地板。)

### 殘差在整個範圍都有用,不是只有極端

把德國殘差切 10 等分,看每格對應的 DK1 實際均價:

| 德國殘差(MW)           | DK1 均價(€/MWh) |
| ---------------------- | --------------- |
| 8,493(最低,**仍為正**) | 14.2            |
| 18,854                 | 45.7            |
| 28,914                 | 77.7            |
| 39,335                 | 111.5           |
| 55,626(最高)           | 166.7           |

**單調爬升,14 → 167,價差 12 倍**——最低那格殘差還是正的。
負殘差(僅 0.8% 時數)只是最戲劇化的尾巴(負電價),不是唯一有用的地方。

### 殘差統計

| 鄰居     | 均值      | 最小        | 負殘差時數 |
| -------- | --------- | ----------- | ---------- |
| 德國     | 33,495 MW | **−13,281** | 0.8%       |
| 瑞典 SE3 | 8,571     | 3,119       | 0%         |
| 瑞典 SE4 | 1,978     | −569        | 0.1%       |

德國殘差最小 −13GW = 風光超過自身用電 → 倒向 DK → **這就是 DK 負電價的來源**。

## 4. 跨境容量 NTC:API 端點的坑

「明天這條電纜排定能送多少電」= 鄰居影響能不能**傳過來**的閘門(容量 0 = 兩邊價格脫鉤)。

**坑**:`query_net_transfer_capacity_dayahead` **只對 DK↔德、DK↔荷發布**。
DK↔挪威/瑞典、DK1↔DK2 內部走**北歐 flow-based 隱式競價**,不在這個端點 → 回空(`NoMatchingDataError`)。

**解法**:換 `query_offered_capacity(from, to, 'A01')`(A01=每日)→ 北歐那幾條就有資料(只到 **2024-10-29**,flow-based 上線後改制)。

**DK 完整拓撲**(已查證,使用者清單正確):

```
DK1 ↔ 德DE_LU / 挪NO_2 / 瑞SE_3 / 荷NL(COBRA) / 英GB(Viking 2023-12) / DK2
DK2 ↔ 德DE_LU / 瑞SE_4 / DK1
```

GB(Viking Link)2023-12 才通,歷史大半空 → 跳過。

**容量本身是弱訊號**(二階閘門):`鄰居殘差(量) × 容量(閘門)` 的互動才是重點。

## 5. 燃料價(Tier-3):來源的現實檢查

要兩個:**天然氣 TTF**(歐洲基準,€/MWh)+ **碳權 EUA**(€/噸 CO2)。

### 實際能用的免費來源(踩過雷後的結論)

| 來源                 | 狀態                                                     |
| -------------------- | -------------------------------------------------------- |
| **yfinance `TTF=F`** | ✅ 天然氣,**完整 2019–2025**(1698 天)                    |
| **yfinance `CO2.L`** | 🟡 碳,值對但**只回到 2021-10-18**(該 ETC 上市日)         |
| investing.com        | ❌ 免費 CSV 下載已鎖進 Pro 付費牆                        |
| Ember                | ❌ 碳價頁併掉了,能免費下載的是電價(來自 ENTSO-E)不是碳價 |
| **EEX 拍賣報告**     | ⚠️ 免費+權威,但**只有拍賣日**(約每週三次),非逐日連續     |
| **Barchart**         | ⚠️ 免費會員可下 EUA 期貨 EOD,但**一天限下載 1 檔**       |

→ 碳價 2019→2021-10 若要補,走 **Barchart**(逐日連續)手動下載一次,丟進 `new_data/fuel/manual/`,
`fuel_prices.py` 的 `load_manual()` 自動吃(認 Date+Price 欄,來源無關)。

### TTF 歷史(驗證正確,教科書級)

| 年       | 均值 €/MWh          |
| -------- | ------------------- |
| 2019     | 14.6                |
| 2020     | 9.6(covid 低點)     |
| 2021     | 47.7                |
| **2022** | **133.3(峰值 339)** |
| 2023     | 41.3                |
| 2024     | 34.6                |

### Leak-safe 規則

燃料是**日資料** → 一天填 24 小時、且用**前一交易日之前的收盤**:
隔日競價約在 noon(D-1)清算,那天燃料還沒結算 → 每小時只能用 **≤ D-2 的收盤**。
實作:`merge_asof(direction='backward')`,cutoff = 交易日 − 2 天(順便填週末/假日)。

### 分區掛載邏輯(`load_duckdb.py` 的 `build_entsoe`)

- 自己的負載 fc + 自己的 SE 鄰居 + 自己的邊界 → 掛該區
- **德國風光 + 德國殘差 = 共用外部驅動 → broadcast 到兩區**
- 燃料價 = 共用,只 join timestamp(不分區)
- 全 leak-free(隔日 / 已 shift),併進 `training` view 仍無 fan-out(131762 列),可訓練 103150 列

## 7. 驗收:接進模型後的結果

### rMAE 進程(LightGBM MAE,單位 €/MWh)

| 階段                              | DK1 MAE   | DK1 rMAE | DK2 MAE   | DK2 rMAE |
| --------------------------------- | --------- | -------- | --------- | -------- |
| 只有本地 5 源                     | 24.36     | 0.76     | 23.48     | 0.70     |
| + ENTSO-E 鄰居風光/負載/NTC       | 23.52     | 0.73     | 21.71     | 0.65     |
| **+ 殘差 + offered-cap + 燃料價** | **17.20** | **0.54** | **17.63** | **0.53** |

**線性模型也跟著跳**(DK1 Ridge 24.75 → 19.90)→ 證明是**真新資訊**,不是樹在自己湊特徵。

### 無 leak 驗證(特徵重要性)

**LightGBM 分裂次數 top:**

```
de_residual_mwh  3764  ← 第一名(德國殘差)
ttf_gas_eur_mwh  3204  ← 天然氣
price_lag24_eur  2738  ← 自迴歸
eua_co2_eur_t    2410  ← 碳價
price_lag168 / nbr_wind / onshore_wind / de_wind / nbr_residual / temperature ...
```

**Ridge 權重 top:** `doy_cos`(季節) −81.9、`price_lag24` 48.5、**`ttf_gas` 31.6**、
`de_residual` 14.6、`nbr_residual` 13.4、`ntc_exp_de` 11.2 …

→ 全部 top 特徵是**自迴歸**(price lag)、**隔日預報**(殘差/風光/負載)、或**燃料價**(已 shift −2 天);
沒有一個是同時刻實測(那些在 `LEAK_COLS` 被排除)。**理論預測會重要的東西,資料證實了 = 最強的反 leak 證據。**
