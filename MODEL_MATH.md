# 模型算式拆解 — 輸入、每個模型、評估指標(用真數字)

> 用資料裡一列真實測試資料把整套算式攤開:**2024-10-09 18:00,DK1,真實電價 y = 122.53 EUR/MWh**。
> 搭配 `new_src/models/baseline.py`(模型)與 `DATA_CATALOG.md`(資料來源)。

---

## 1. 輸入 x 長什麼樣

模型的「一筆輸入」= 一個 **31 維數字向量**。那列真的長相(節選):

```
時間: 2024-10-09 18:00       真實電價 y = 122.53 EUR/MWh  ← 要預測的答案

x = [
  日曆:   hour=18, dow=2(週三), month=10, is_weekend=0,
          hour_sin=-1.0, doy_sin=-0.987, ...        (時間的正弦編碼)
  天氣:   wind_speed_100m=23.4, temperature_2m=14.4,
          cloud_cover=70, shortwave_radiation=23, ...
  隔日出力預測: offshore_wind_da_mwh=616, onshore_wind_da_mwh=364,
               solar_da_mwh=6.4        (Energinet 給的,leak-free)
  lag:    price_lag24_eur=144.0,  price_lag168_eur=117.0,
          load_lag24_mwh=2993,  residual_lag24_mwh=1170
]
```

每個模型做的事都一樣:吃 x,吐一個 ŷ,希望貼近 122.53。差別只在 **x → ŷ 的算式**。

---

## 2. 每個模型的算式(用這列的真數字)

### naive(地板)

最笨、零學習:

```
ŷ_naive = 24 小時前同一小時的價 = price_lag24_eur = 144.00
```

即「明天晚 6 點 = 今天晚 6 點」。這列真值 122.53 → 猜錯 |122.53 − 144.00| = **21.47**。
存在的唯一意義:**當一把尺**,模型贏不過它就是白做。

### Ridge / Lasso(線性)

一條加權總和:

```
ŷ = b + w₁·x₁ + w₂·x₂ + ... + w₃₁·x₃₁
```

`b` = 基準,`wᵢ` = 每個特徵的權重(重要性 + 方向)。這列 Ridge 實際算出:

```
ŷ = 97.88(基準)
    + 73.09 × price_lag24     ← 昨天的價,權重最大(主導)
    − 11.22 × onshore_wind    ← 風越多 → 價越低(負號 = 供需)
    −  8.65 × is_weekend      ← 週末更便宜
    + 23.31 × price_lag168    ← 上週同時段
    + ...(其餘 27 個)
    = 145.01
```

**「訓練」= 找出那 31 個 wᵢ**,方法是最小化:

```
Ridge:  minimize   Σ(y − ŷ)²  +  α·Σwᵢ²      ← 後項懲罰,逼權重別太大
Lasso:  minimize   Σ(y − ŷ)²  +  α·Σ|wᵢ|     ← 絕對值,把沒用的 wᵢ 壓成 0
```

前項「要準」,後項(α)「別過度依賴任何特徵」——這就是 Ridge/Lasso 跟純線性回歸(OLS)的差別,讓它對沒看過的未來也穩。Lasso 的絕對值懲罰會把部分權重直接歸零 = **自動挑特徵**,所以叫 LEAR。

權重是有物理意義的:**風力預測權重為負**(風多→電多→價跌)、**週末為負**(不開工)。數學自己從資料長出來的。

### LightGBM(樹)

不是一條式子,是一堆小決策樹相加:

```
ŷ = 基準 + f₁(x) + f₂(x) + ... + f₃₀₀₀(x)
```

每棵 fₖ 是一串 yes/no 問題:

```
        price_lag168 > 100 ?
        ├─ 是 → wind_da > 500 ?
        │        ├─ 是 → +3.2
        │        └─ 否 → +8.7
        └─ 否 → hour > 17 ? ...
```

每棵樹只修前面所有樹「還沒修好的殘差」(gradient boosting,梯度提升)。資料裡它最愛切的是
`price_lag168`(1022 次)、`price_lag24`(928)、`doy_sin`、`dow`——**跟線性模型倚重的幾乎一樣**。

**這解釋了為什麼它只贏線性一點點**:樹的強項是非線性 + 交互作用,但這問題的關係大多單調
(價越高→明天越高、風越多→越低),線性已吃掉,樹沒剩多少可撿。

## 3. 怎麼評估好壞(用真數字)

先看**這一列**:

| 模型  | ŷ      | 誤差 y−ŷ | \|誤差\| |
| ----- | ------ | -------- | -------- |
| naive | 144.00 | −21.47   | 21.47    |
| Ridge | 145.01 | −22.48   | 22.48    |

**這列 naive 反而比 Ridge 準!** 單列毫無意義 → 對**全部 10920 列測試資料**取平均:

```
MAE  = (1/N)·Σ|yᵢ − ŷᵢ|            平均差幾塊歐        → naive 32.04, Ridge 24.64
RMSE = √( (1/N)·Σ(yᵢ − ŷᵢ)² )      先平方再開根,重罰大失誤 → naive 48.91, Ridge 36.58
rMAE = MAE_模型 / MAE_naive        = 24.64/32.04 = 0.77
```

- **MAE**:10920 個 |誤差| 的平均。純「平均差多少歐」。
- **RMSE**:差 100 的一列平方後貢獻 10000 → 幾個大爆走就把它拉高。RMSE 比 MAE 高 = 有些時刻錯很大(尖峰)。
- **rMAE**:模型 MAE ÷ naive MAE。0.77 = 比抄昨天少錯 23%。**三個都越小越好,rMAE < 1 才及格。**

### 目前結果

|         | naive | Ridge | Lasso | LightGBM | 最佳 rMAE |
| ------- | ----- | ----- | ----- | -------- | --------- |
| **DK1** | 32.04 | 24.64 | 24.55 | 24.36    | 0.76      |
| **DK2** | 33.37 | 24.80 | 24.71 | 23.48    | 0.70      |

(MAE,單位 EUR/MWh。三模型全贏地板 24–30%。)

## 4. 還有沒有更適合的模型

**「梯度下降(gradient descent)不是模型,是訓練方法」** —— 用梯度下降訓練的東西通常是**神經網路(DNN)**。
在 EPF 領域,DNN 是**兩大標竿之一**:

> EPF 最權威 benchmark(Lago 等人 2021)拿 **LEAR(=Lasso)vs DNN** 對打,結論:兩個都很強、
> 誰也沒完全贏誰,**兩個做 ensemble(平均)最好**。

完整地圖:

| 家族           | 代表                                          | 說明                       |
| -------------- | --------------------------------------------- | -------------------------- |
| 線性           | OLS / Ridge / **Lasso(LEAR)** / ElasticNet    | LEAR 是參考標竿,已有       |
| 樹             | **LightGBM** / XGBoost / RandomForest         | 已有                       |
| 神經(梯度下降) | **DNN**(前饋)/ LSTM,GRU / TCN / N-BEATS / TFT | DNN 最常見;LSTM/TFT 抓時序 |
| 傳統時序       | ARIMA / SARIMAX / ARX                         | 老派單變數,通常打不贏上面  |

### 下一步該做什麼(修正:資料不是唯一槓桿)

現在有**兩條沒試過的路**,不確定哪條贏:

1. **模型結構(不用新資料,今天就能做)**
   - **per-hour**:24 小時各訓一個模型(LEAR/DNN 標準做法)
   - **ensemble**:LEAR + LightGBM +(之後)DNN 平均——文獻說最穩
   - **target asinh 轉換**:壓尖峰肥尾
2. **新資料(等 ENTSO-E token)**:Tier-2 鄰居 + Tier-3 天然氣——**主要救 2022 那種極端 regime**,
   對「一般日子 MAE」幫助可能沒那麼大。

### 需不需要跨區資訊?很可能不需要(就能拿到不錯結果)

模型實際最倚重的是 `price_lag24/168`、`doy`、`dow`、`onshore_wind`、`load_lag`——**全是本地 + 自迴歸**。
跨區與燃料價主要在**尖峰和跨年乾濕 regime** 才發威。要拿「不錯的日常 MAE」,自迴歸 + 本地天氣/出力預測
已吃掉大半。沒跨區也跑得好,完全合理。

**建議順序**:先試 **per-hour + ensemble(LEAR+LGBM)**(免費、文獻證實有效),再看要不要加 DNN;
Tier-2 資料等 token 到當「加分題」,不是必需品。
