## 1. 最重要的三件事

### ① λ 從 0.037 改成 0.004(舊值高估 8 倍)

舊的 0.037 是**單變數 OLS**(價格 ~ 本地殘餘負載,只控燃料)。問題是**混淆**:
丹麥風大的日子北德通常也風大(同一片天氣系統),所以本地殘餘低時德國殘餘也低、價格也低,
但**不是因為丹麥本地的供需**。單變數把德國的效果全記到本地頭上。

新的 0.0042 是**多變數偏導數** ∂p/∂x,控制德/瑞殘餘、燃料、邊界、日曆後求值。
這才是 λ 該回答的問題——「我的電池多買 1 MW,價格漲多少」,那個實驗裡鄰國和燃料**都不變**。

程式:`new_src/agents/fringe.py` 的 `structural_lambda()`。**這是 repo 裡第一次有能重跑產出 λ 的程式碼**
(0.037 以前只活在文件的一句話裡,沒有出處)。

### ② 丹麥現況下,多 agent 機制加了零

|                                    | 佔單顆基準 |
| ---------------------------------- | ---------- |
| 10 家競爭 + 預測誤差(v2.2 λ=0.004) | **93%**    |
| 單顆 LightGBM(完全沒有競爭)        | **93%**    |

`λ×總淨量 = 0.004 × 10MW = 0.04 €/MWh`,在 €60 的價格上是 0.07%。撼動不了任何東西。
**這不是 bug,是丹麥這個尺度的實況**(全國最大併網電池案例才 10–30 MW)。

三把尺也印證:10 MW 時 C = N = M 塌成一點,勾結空間 **0.0%**。
**tacit collusion 這個題目在丹麥現況下不存在**——這是乾淨的負面結論,不是失敗。

### ③ 預測優勢在競爭下被**放大** 6 倍(這是你研究問題的答案)

10 家同體量,只有預測品質不同(3 LightGBM / 3 Ridge / 4 naive-24h):

| 體量      | LightGBM | naive-24h | 優勢倍數 |
| --------- | -------- | --------- | -------- |
| 10 MW     | 93%      | 78%       | 1.18×    |
| 1 GW      | 83%      | 65%       | 1.28×    |
| **10 GW** | **39%**  | **6%**    | **6.3×** |

**競爭越兇,預測優勢越值錢——不是被稀釋,是被放大。**
直覺:競爭把價差壓平之後,只有看得準的人抓得到剩下的套利窗口;看不準的人在錯的時段
進出,被出清價懲罰。10 GW 下 naive 幾乎賺不到錢(6%),LightGBM 仍有 39%。

## 2. 觀念釐清(這幾個之前一直卡住)

### `act` 是什麼

`act` = 歷史上真實的 day-ahead 出清價。**它是一個結果,不是一個輸入。**
裡面已經攪在一起的東西:燃料成本、每個發電商的加價策略(市場力!)、
德國/挪威/瑞典透過連接線壓進來的價、風光出力、需求。**沒辦法從 act 裡把任何一項單獨拆出來。**

你的公式 `p = act + λ·Q` 的意思是:「拿歷史上真的發生過的價格,把我想像出來的電池隊的影響加上去。」

### 市場力是什麼

**不是「互相 bidding」這個動作,是「有能力靠報高價賺到錢」。**
小機組報高價沒人理它(不被調度)= 沒有市場力;pivotal 機組(沒有它就湊不滿需求)報高價,
市場**必須付** = 有市場力。衡量方式是「價格 vs 邊際成本」或「實際價 vs 大家都照成本報價的價」——
**兩種都需要成本資料**。

丹麥實況:DK1 對外有 6,000+ MW 連接線容量、本地尖峰才 3.5 GW,進口隨時取代本地機組,
所以丹麥本地幾乎沒人是 pivotal。

### 為什麼不能宣稱市場力(循環)

**這裡有兩個不同的病,可治程度完全不同:**

| 病       | 內容                                                  | 治得好嗎                                         |
| -------- | ----------------------------------------------------- | ------------------------------------------------ |
| **混淆** | 殘差裡混著德國、燃料、本地需求,分不清誰是誰           | **治得好**——加控制變數、工具變數。§1① 就是治這個 |
| **循環** | p₀ 是**用 act 擬合出來的**,act 減掉它必然是零均值噪音 | **加再多變數都治不好**                           |

循環的比喻:想知道一家店有沒有超收,要查它的**進貨成本**。如果把「合理價」定義成
「這家店的平均售價」,那它在數學上**永遠不可能超收**,就算賣十倍也一樣。

**所以本模型可以做相對比較**(A agent vs B agent,兩者面對同一個 p₀)**,不能宣稱絕對市場力**。
要打破循環,p₀ 必須從成本結構推出來(燃料÷效率 + 碳價×排放強度 + 機組容量),不能從價格擬合。
**而你的研究問題是相對的,所以不需要打破它。** 這是這次最重要的策略決定。

### 為什麼 λ(x) 曲線沒有接進模擬器

兩個理由:

1. **λ(x) 是局部斜率**。`p = p₀ + λ(x)·Q` 只在 Q 小到不會沿曲線跑遠時成立。
   GW 級車隊會推動 x 幾百 MW,要改成 `p = p₀(x+Q)`(非線性,會動到 `perfect()` 的 LP)。
2. **它 condition 在小的那個維度上**。λ 隨**燃料** regime 變 4.4 倍(0.010/0.019/0.044),
   隨**殘餘負載**只變 2.4 倍。只做 λ(x) 會被問「你為什麼忽略更大的那個維度」——
   從「誠實的粗糙」變成「聲稱精確但只做了一半」。

## 5. 過程中發現的錯誤(避免重蹈)

| 錯誤                                   | 怎麼發現的                                                                                                                                                             |
| -------------------------------------- | ---------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **分箱差分估 λ(x) 有 8/40 箱是負斜率** | 供給曲線不可能向下傾斜 → 改用保序迴歸。我第一次給的 5 個分位點表是 cherry-pick 的                                                                                      |
| **稀缺尾巴 5.3× 是尖刺假象**           | 保序解是階梯函數,微分後尖刺。平滑後是 **2.4×**。**如果你筆記裡有 5.3,改掉**                                                                                            |
| **雲圖混合 regime**(你抓到的)          | 把 2020 年 gas €4 和 2022 年 €339 攤在同一張圖,等於宣稱「x=1000 時價格可能 €7 到 €770」。那是兩個世界,不是同一個世界的變異                                             |
| **偏導數第一次跑出 0.0004**            | 共線性:x = 負載−風−光,三者的預測值都在特徵集裡,「固定它們只動 x」是矛盾的反事實                                                                                        |
| **有限差分步長 50MW 得到 0.0000**      | 樹是階梯函數,42% 的小時落在同一葉子。要用 300MW                                                                                                                        |
| **v4 拿價格反推風力**                  | 那會把「風與價負相關」這個**待驗證的性質**變成建構出來的假設,而外部性的符號正是該節結論。改用真實風力預測後,純風商從 12,208 降到 4,140 €/MW/週(容量因數從 100% 變 41%) |
| **spark spread 修不了斜率**            | 扣掉的燃料成本在一小時內是常數,常數不影響對 x 的偏微分。它修的是**水平**不是**斜率**                                                                                   |

還有一個觀念錯誤值得記:**23% 未解釋不是市場力**。同樣 4 個變數只改函數形式,
線性 R²=0.773(未解釋 23%)→ LightGBM R²=0.971(未解釋 3%)。**零新變數、零市場力假設,
20 個百分點就消失了**——那是「用直線配一條彎的曲線」的誤差。所以 23% 是上界,不是市場力。

## 7. 論文可以怎麼講(草稿方向)

一個誠實且站得住的故事線:

1. **丹麥現況下儲能沒有市場力**——10 MW 車隊,C=N=M,勾結空間 0.0%。有數字、有三把尺當證據。
2. **而且過去的估計方法會高估**——單變數 λ 高估 8 倍,因為捕捉的是丹麥/德國的天氣相關性
   而非本地供需。這是一個方法論貢獻。
3. **真正的瓶頸是預測不是競爭**——預測誤差成本 7pp >> 價格衝擊成本 0pp。
4. **而預測優勢在競爭下被放大**——這是主結果,而且反直覺(一般會猜競爭把大家拉平)。
5. **門檻在 GW 級**——給政策含意:丹麥現在不需要擔心,儲能規模到 GW 才需要監管關注。

**範圍要誠實揭露的**:p₀ = act 的循環(所以只做相對比較)、λ 是觀察性估計非因果識別、
10GW 是前瞻情境非丹麥現況。
